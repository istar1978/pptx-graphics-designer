# 网络拓扑图生成指南

## 概述

`network_topology.py` 是一个专业的 PowerPoint 网络拓扑图生成器，支持创建 Visio 风格的企业网络架构图。

## 主要功能

### 1. 标准网络设备图标

| 设备类型 | 图标样式 | 颜色 | 说明 |
|---------|---------|------|------|
| `router` | 椭圆形带⇄符号 | 蓝色 (#0099CC) | 路由器/核心路由 |
| `switch` | 椭圆形带⇅符号 | 蓝色 (#0099CC) | 交换机/核心交换 |
| `firewall` | 立方体 | 橙色 (#FF6600) | 防火墙/安全设备 |
| `load_balancer` | 六边形 | 紫色 (#9900CC) | 负载均衡器 |
| `server` | 立方体带≡符号 | 灰色 (#666666) | 应用服务器 |
| `database` | 圆柱体 | 绿色 (#00994C) | 数据库服务器 |
| `storage` | 圆柱体 | 蓝色 (#006699) | 存储设备 |
| `pc` | 矩形 | 浅蓝色 (#6699CC) | 台式机/终端 |
| `laptop` | 矩形 | 浅蓝色 (#6699CC) | 笔记本电脑 |
| `phone` | 圆角矩形 | 紫色 (#9966CC) | 手机/移动设备 |
| `cloud` | 云朵形 | 蓝色 (#4285F4) | 云服务 |
| `internet` | 椭圆形带🌐符号 | 蓝色 (#4285F4) | 互联网接入 |
| `datacenter` | 矩形 | 灰色 (#666666) | 数据中心 |

### 2. 网络区域容器

| 区域类型 | 背景色 | 边框色 | 用途 |
|---------|--------|--------|------|
| `internet` | 浅蓝 (#F5FAFF) | 蓝色 (#4285F4) | 互联网接入区 |
| `dmz` | 浅橙 (#FFF9F5) | 橙色 (#FF9900) | DMZ 隔离区 |
| `office` | 浅绿 (#F5FFF5) | 绿色 (#00994C) | 办公网络区 |
| `datacenter` | 浅紫 (#FAF5FF) | 紫色 (#9900CC) | 数据中心区 |
| `security` | 浅红 (#FFF5F5) | 红色 (#CC0000) | 安全管理中心 |
| `core` | 浅蓝 (#F5FAFF) | 蓝色 (#0066CC) | 核心交换区 |
| `custom` | 浅灰 (#F5F5F5) | 灰色 (#969696) | 自定义区域 |

### 3. 连接线

- **实线** - 默认连接类型
- **虚线** - 用于表示逻辑连接或专线（设置 `dashed: true`）
- **带标签** - 可在连接线上添加文字标签（如"MPLS"、"专线"）
- **自定义颜色** - 支持自定义线条颜色
- **自定义宽度** - 支持调整线条粗细

## 使用方法

### 基本命令

```bash
python scripts/network_topology.py \
  --input <配置文件.json> \
  --output <输出文件.pptx> \
  --title <图表标题>
```

### 参数说明

| 参数 | 必需 | 说明 |
|------|------|------|
| `--input, -i` | 是 | JSON 配置文件路径 |
| `--output, -o` | 是 | 输出 PPTX 文件路径 |
| `--title, -t` | 否 | 图表标题（默认："网络拓扑图"） |
| `--template` | 否 | 使用模板：`reference`（参考图）或 `custom`（自定义） |

### JSON 配置结构

```json
{
  "title": "图表标题",
  "zones": [
    {
      "id": "区域唯一 ID",
      "type": "区域类型",
      "title": "区域显示标题",
      "x": X 坐标（英寸）,
      "y": Y 坐标（英寸）,
      "width": 宽度（英寸）,
      "height": 高度（英寸）
    }
  ],
  "devices": [
    {
      "id": "设备唯一 ID",
      "type": "设备类型",
      "x": X 坐标（英寸）,
      "y": Y 坐标（英寸）,
      "width": 宽度（英寸，默认 0.8）,
      "height": 高度（英寸，默认 0.6）,
      "label": "设备标签"
    }
  ],
  "connections": [
    {
      "from": "起始设备 ID",
      "to": "目标设备 ID",
      "label": "连接标签（可选）",
      "color": "线条颜色 RGB（可选）",
      "width": 线条宽度（可选，默认 1.5）,
      "dashed": 是否虚线（可选，默认 false）
    }
  ]
}
```

## 坐标系统

- 幻灯片尺寸：13.333 英寸 × 7.5 英寸（16:9 比例）
- 原点 (0, 0)：幻灯片左上角
- X 轴：从左到右递增
- Y 轴：从上到下递增

### 推荐边距

- 左边距：0.5 英寸
- 右边距：0.5 英寸
- 上边距（含标题）：1.0 英寸
- 下边距：0.5 英寸
- 可用宽度：约 12.3 英寸
- 可用高度：约 6.5 英寸

## 布局建议

### 1. 区域布局

```
┌─────────┐ ┌───────── ┌─────────┐
│ 云端    │ │ 总部    │ │ 分支 1  │
│ 0.5,0.8 │ │ 4.5,0.8 │ │ 9.5,0.8 │
│ 3.5×2.5 │ │ 4.5×4.5 │ │ 3.5×2.0 │
└─────────┘ └─────────┘ └─────────┘
            ┌─────────┐
            │ 分支 2  │
            │ 9.5,3.2 │
            │ 3.5×2.0 │
            └─────────┘
```

### 2. 设备间距

- 同区域设备水平间距：0.2-0.3 英寸
- 同区域设备垂直间距：0.3-0.5 英寸
- 设备与区域边界：至少 0.2 英寸

### 3. 典型布局模式

#### 三层架构
```
        [核心层]
           │
      ┌────┴────┐
      │         │
   [汇聚 1]  [汇聚 2]
      │         │
   ┌──┴──┐  ┌──┴──┐
   │     │  │     │
 [接入 1]...[接入 N]
```

#### 星型拓扑
```
        [核心]
       /  │  \
      /   │   \
   [分支 1]...[分支 N]
```

#### 链式拓扑
```
[总部]──[汇聚]──[接入 1]──[接入 2]
```

## 最佳实践

### 1. 设计原则

- **清晰第一** - 避免过度拥挤，保持足够间距
- **逻辑分组** - 相关设备放在同一区域
- **颜色一致** - 同类型设备使用相同颜色
- **标签简洁** - 2-4 个中文字符或 1-2 个英文单词

### 2. 性能考虑

- 单个幻灯片建议不超过 30 个设备
- 连接线避免过多交叉
- 复杂拓扑可拆分为多个幻灯片

### 3. 可访问性

- 确保颜色对比度足够（至少 4.5:1）
- 不要仅依赖颜色传达信息
- 添加文字标签说明关键连接

## 故障排除

### 问题 1：设备重叠

**原因**：坐标计算错误或间距过小

**解决**：
1. 检查设备坐标和尺寸
2. 增加设备间距（建议 0.2-0.3 英寸）
3. 调整区域大小

### 问题 2：连接线混乱

**原因**：设备位置不合理或连接过多

**解决**：
1. 重新排列设备位置
2. 使用正交布局（水平/垂直对齐）
3. 考虑拆分拓扑

### 问题 3：文字溢出

**原因**：标签过长或字体过大

**解决**：
1. 缩短标签文字
2. 调整设备尺寸
3. 减小字体大小（不低于 8pt）

### 问题 4：区域边界不清晰

**原因**：背景色太浅或边框太细

**解决**：
1. 使用更深的背景色
2. 增加边框宽度（2-3pt）
3. 增加区域间距

## 输出验证

生成 PPTX 后，建议检查：

- [ ] 所有设备都正确显示
- [ ] 连接线方向正确
- [ ] 标签文字清晰可读
- [ ] 区域分组合理
- [ ] 配色协调一致
- [ ] 无设备溢出边界
- [ ] 整体布局平衡美观
